## 0x01 模幂运算的两种遍历方向

在 RSA 等公钥密码体制中，核心运算是模幂 $c^d \bmod n$。根据遍历私钥比特位的方向不同，存在两种经典算法。

### 从高到低（Left-to-Right）

#### 核心直觉

类似我们从左到右读一个十进制数。以 523 为例：

1. 看到 **5**：当前值 = 5
2. 看到 **2**：先乘以基数 10，再加新位 → $5 \times 10 + 2 = 52$
3. 看到 **3**：再乘以 10，加新位 → $52 \times 10 + 3 = 523$

映射到二进制幂运算：每向右走一位，指数"乘以 2"对应底数的**平方**，指数"加上 $d[i]$"对应底数**乘以 $c$**。

#### 伪代码

```
s := 1
for i := w - 1 down to 0 do
    s := s * s mod n         // 先平方
    if d[i] == 1 then
        s := s * c mod n     // 后乘底数 c
    end if
end for
return s
```

#### 实例追踪：计算 $c^5$（$5 = 101_2$，$d[2]=1, d[1]=0, d[0]=1$）

| 步骤 | 当前位 | 操作 | 状态 |
|------|--------|------|------|
| 初始 | - | $s = 1$ | $s = 1$ |
| $i=2$ | $d[2]=1$ | $s = 1^2 = 1$，再 $s = 1 \times c = c$ | $s = c$ |
| $i=1$ | $d[1]=0$ | $s = c^2$，跳过乘法 | $s = c^2$ |
| $i=0$ | $d[0]=1$ | $s = (c^2)^2 = c^4$，再 $s = c^4 \times c = c^5$ | $s = c^5$ |

**数学规律**：循环每一步中，$s$ 计算的都是 $c^{\lfloor d / 2^i \rfloor}$。当 $i$ 降到 0 时，$\lfloor d / 2^0 \rfloor = d$ 本身。

### 从低到高（Right-to-Left）

#### 核心原理

将指数 $d$ 做二进制展开：

$$d = d_0 \cdot 2^0 + d_1 \cdot 2^1 + d_2 \cdot 2^2 + \dots$$

则幂运算拆为乘积：

$$c^d = c^{d_0 \cdot 2^0} \cdot c^{d_1 \cdot 2^1} \cdot c^{d_2 \cdot 2^2} \dots$$

维护一个底数自乘变量 $A$，每轮平方依次变为 $c^1, c^2, c^4, c^8 \dots$。若当前位 $d[i] = 1$，就把 $A$ 乘入累乘器 $R$。

#### 伪代码

```
R := 1
A := c
for i := 0 to w - 1 do
    if d[i] == 1 then
        R := R * A mod n
    end if
    A := A * A mod n
end for
return R
```

#### 实例追踪：计算 $c^5$（$5 = 101_2$，$d[0]=1, d[1]=0, d[2]=1$）

| 步骤 | 当前位 | R 的变化 | A 的变化 |
|------|--------|----------|----------|
| 初始 | - | $R = 1$ | $A = c$ |
| $i=0$ | $d[0]=1$ | $R = 1 \times c = c$ | $A = c^2$ |
| $i=1$ | $d[1]=0$ | $R$ 不变 = $c$ | $A = (c^2)^2 = c^4$ |
| $i=2$ | $d[2]=1$ | $R = c \times c^4 = c^5$ | $A = (c^4)^2 = c^8$ |

### 两种算法对比

| 特征 | 从高到低 (Left-to-Right) | 从低到高 (Right-to-Left) |
|------|--------------------------|--------------------------|
| 核心直觉 | "已有结果平方，再看加不加新位" | "底数不断平方，按位决定要不要" |
| 循环方向 | $w-1 \to 0$ | $0 \to w-1$ |
| 状态变量 | 1 个核心变量 $s$ 演变 | 2 个独立变量：累乘器 $R$ 和底数权重 $A$ |

## 0x02 时间侧信道泄漏

### 漏洞根源

两种算法都存在 `if d[i] == 1` 的分支判断：

```
if d[i] == 1 then
    m[i] := s[i + 1] * c mod n    // 大数模乘，耗时长
else
    m[i] := s[i + 1] * 1           // 乘以 1，几乎不耗时
end if
```

**根本原因**：

1. **运算量极度不对称**：$d[i] = 1$ 时需执行大数模乘（数千位的乘法非常吃 CPU 周期），$d[i] = 0$ 时仅乘以 1，耗时可忽略
2. **时间可观测**：每轮的平方耗时是固定的，总耗时的长短完全取决于 `if` 走哪边

攻击者多次发起加密请求，用高精度时钟测量总执行时间，通过统计学方法即可逐位还原私钥。

### 从低到高算法的防御优势

Right-to-Left 算法中，$R$ 的乘法和 $A$ 的自平方在数学上相互独立，适合改为**恒定时间算法**：引入哑变量 `R_dummy`，在 $d[i] = 0$ 时把结果乘到哑变量里，使每轮耗时完全一致。

## 0x03 ECC 标量乘法：Double-and-Add

### 数学原理

在椭圆曲线密码学中，核心运算是标量乘法 $Q = k \cdot P$。设 $n$ 位私钥标量 $k$ 的二进制形式为：

$$k = (k_{n-1}k_{n-2}\dots k_0)_2 = \sum_{i=0}^{n-1} k_i 2^i$$

算法通过从高位到低位扫描 $k$，利用群的阶乘倍性质，将复杂度由 $O(k)$ 降至 $O(\log k)$。

### 算法流程

1. **初始化**：令累加点 $Q = \mathcal{O}$（无穷远点 / 群零元）
2. **高位到低位循环**：遍历 $i$ 从 $n-1$ 降至 0
3. **无条件倍点 (Double)**：每轮执行 $Q = 2 \cdot Q$
4. **条件点加 (Add)**：若 $k_i = 1$，则 $Q = Q + P$；否则跳过

### 算力开销

| 密钥位 $k_i$ | 执行的操作 | 代价（雅可比坐标系） |
|--------------|-----------|---------------------|
| 0 | 仅倍点 | $1 \times \text{ECDBL}$ |
| 1 | 倍点 + 点加 | $1 \times \text{ECDBL} + 1 \times \text{ECADD}$ |

### 示例：计算 $5P$（$k = 5 = 101_2$）

| 步骤 | 位 | 操作 | 状态 |
|------|-----|------|------|
| 初始 | - | $Q = \mathcal{O}$ | $Q = \mathcal{O}$ |
| Bit 2 | 1 | Double: $Q = 2\mathcal{O} = \mathcal{O}$，Add: $Q = \mathcal{O} + P = P$ | $Q = P$ |
| Bit 1 | 0 | Double: $Q = 2P$，跳过 Add | $Q = 2P$ |
| Bit 0 | 1 | Double: $Q = 4P$，Add: $Q = 4P + P = 5P$ | $Q = 5P$ |

## 0x04 ECC 侧信道攻击与防御

### 标准算法的安全隐患

标准 Double-and-Add 在现代密码学工程中**禁止直接使用**，缺陷在于执行流的非对称性。

由于 ECDBL 和 ECADD 的底层有限域运算组合不同，它们的功耗轨迹和执行耗时有显著差异：

```
密钥位 1:  [  ECDBL  ][  ECADD  ]  --> 双峰 / 长时耗
密钥位 0:  [  ECDBL  ]             --> 单峰 / 短时耗
```

攻击者通过示波器捕获芯片运算时的电流波动（SPA 简单功耗分析），观察到「长-短-长」或「高能量-低能量」的规律，即可直接反推私钥的二进制序列。

### 防御方案 A：Double-and-Add-Always

**原理**：不论 $k_i$ 是 0 还是 1，都同时计算 ECDBL 和 ECADD。

```
Q[0] = 2 * Q[0]
Q[1] = Q[0] + P
Q[0] = Q[k_i]     // 若 k_i 为 0，抛弃 Q[1] 的计算结果
```

**优缺点**：抗简单侧信道攻击，但引入了安全哑操作，易受**错误注入攻击**——破坏哑操作若结果不变，说明该位是 0。

### 防御方案 B：蒙哥马利梯子（Montgomery Ladder）

**原理**：维护两个点的关系，每一轮都在并行执行一次倍点和一次点加，无哑操作。

```
If k_i == 0:
    R[0] = 2 * R[0],  R[1] = R[0] + R[1]
If k_i == 1:
    R[1] = 2 * R[1],  R[0] = R[0] + R[1]
```

**优势**：天然具备极高的抗 SPA、抗时序攻击和抗错误注入攻击能力，是目前主流硬件安全芯片和加密库（如 Curve25519）的首选实现方案。

## 0x05 核心考点

- **模幂运算**：Left-to-Right 和 Right-to-Left 两种遍历方式的原理与实现
- **时间侧信道**：分支判断导致运算量不对称，攻击者可通过计时还原私钥
- **ECC Double-and-Add**：标量乘法的基本流程与复杂度优化
- **SPA 功耗分析**：ECDBL 和 ECADD 的功耗差异泄露密钥位信息
- **恒定时间防御**：Double-and-Add-Always 与 Montgomery Ladder 的权衡
- **错误注入攻击**：哑操作方案的固有缺陷
