0x01 IDA Pro 静态分析

反编译与视图切换

快捷键 功能 使用场景
F5 反编译为 C 伪代码 快速理解程序逻辑，查看函数调用和条件判断
Space 汇编流程图 / 伪代码切换 流程图可清晰展示 if-else、switch 跳转逻辑
TAB Hex View 切换 查看和编辑底层十六进制机器码

显示指令地址

IDA 默认不在汇编代码左侧显示内存地址，需要手动开启：

Options → General → Line prefixes 勾选

开启后每行指令前会显示对应的内存地址（如 0x40082b），这是联动 GDB 下断点的关键。

静态数据溯源

在伪代码中双击未初始化的全局变量，IDA 会自动跳转到对应的 .data（数据段）或 .rodata（只读数据段），可直接提取硬编码字符串和敏感数据。

常见数据段：

  • .data：已初始化的全局变量和静态变量
  • .rodata：只读数据，如字符串常量
  • .bss：未初始化的全局变量

二进制打补丁（Patch）

IDA 可以直接修改可执行文件的指令和机器码：

修改汇编指令

  1. 选中目标汇编指令
  2. Edit → Patch program → Assemble...
  3. 输入新的汇编指令（如将 jnz 改为 jz，或用 nop 抹除指令）

修改机器码

  1. 按 TAB 切换到 Hex View
  2. 按 F2 进入编辑模式
  3. 直接修改十六进制值（如 75 改为 74 或 90）
  4. 再按 F2 保存

导出修改后的程序

Edit → Patch program → Apply patches to input file...

勾选 Backup 选项，将修改写入新的可执行文件。

0x02 GDB 动态调试

基础操作

命令 简写 功能
gdb ./程序名 - 启动 GDB 并挂载程序
run r 运行程序，可带参数如 r < input.txt
continue c 从断点处继续执行，直到下一个断点或程序结束
delete d 删除断点，d 删除全部，d 1 删除编号 1 的断点
info registers i r 查看当前所有寄存器的值

断点管理：带 * 与不带 * 的区别

这是 GDB 调试中最容易混淆但最重要的概念之一。

带星号 *（汇编级精确打击）

  • GDB 将后面的值严格作为绝对内存地址
  • 程序会精确停在指定地址的汇编指令处
  • CTF 最常用：无源码逆向时，根据 IDA 中的地址直接下断

如果不带 * 输入 b 0x40082b，GDB 可能将其误解为”第 4196395 行源代码”而报错。

不带星号（源码级符号调试）

  • 依赖程序的调试符号（Symbols）
  • GDB 会跳过函数序言（push rbp 等准备工作），停在第一句 C 业务代码处
  • 适用于有源码的日常开发，但在剥离符号表（Stripped）的 CTF 程序中往往失效

寄存器劫持（核心技巧）

这是绕过随机数陷阱、强制跳转 switch 分支的核心技术。

原理：x86 架构中，eax 通常用于存放函数返回值或临时计算结果。在 mov [rbp+var_34], eax（将计算结果写入内存）之前修改 eax，程序后续的 cmp 和跳转表就会按修改后的值执行。

应用场景：

  • 绕过 rand() 随机数陷阱，强制进入特定 switch 分支
  • 强行绕过本地注册码校验（将校验结果寄存器改为 1）
  • 在加壳程序的 OEP 处停下进行内存 dump

输出缓冲区注意事项

C 语言的 printf 在遇到 \n（换行符）或程序正常退出时才将内容推送到终端。如果在调试时看到输出被截断，往往是因为程序被断点挂起，缓冲区尚未刷新。

解决方法：删除断点，让程序正常跑完并退出，强制刷新缓冲区。
